## Overview

Dwex provides a powerful, decorator-based routing system that makes defining API endpoints intuitive and type-safe. Routes are defined using method decorators that correspond to HTTP verbs.

## Routing Basics

Routes are defined in controllers using decorators:

```typescript
@Controller("posts")
export class PostController {
	@Get() // GET /posts
	findAll() {
		return [];
	}

	@Get(":id") // GET /posts/:id
	findOne(@Param("id") id: string) {
		return { id };
	}

	@Post() // POST /posts
	create(@Body() data: CreatePostDto) {
		return { id: 1, ...data };
	}

	@Delete(":id") // DELETE /posts/:id
	remove(@Param("id") id: string) {
		return { deleted: true };
	}
}
```

## Features

**HTTP Methods** - Decorators for GET, POST, PUT, PATCH, DELETE, OPTIONS, HEAD, and ALL

**Route Parameters** - Extract dynamic segments from URLs with `@Param()`

**Query Strings** - Access query parameters with `@Query()`

**Request Body** - Parse JSON bodies with `@Body()`

**Headers & Cookies** - Access headers with `@Headers()` and cookies with `@Cookies()`

## Route Composition

Combine controller and method paths:

```typescript
@Controller("api/v1/users") // Base path
export class UserController {
	@Get(":id/posts") // Results in: GET /api/v1/users/:id/posts
	getUserPosts(@Param("id") id: string) {
		return [];
	}
}
```
